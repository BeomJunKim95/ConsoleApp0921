메서드 오버라이드(ToString, Equals 재정의), 연산자 오버로드, 클래스간의 형변환 , 메서드(call by value, call by ref, out) , params 키워드
연산자 오버로드( +, ++, >, < )
overload - 다형성,  메서드 중복(이게 일반적)
override - 상속성 재정의 ( virtual , override, abstract  했을때)

추상클래스 : new불가능
	    추상매서드가 하나 이상인 클래스
 	    대부분 가장 상위클래스를 추상클래스로 정의 
	    자식클래스 ---> 부모클래스 ----> 추상클래스(최상위)
	    버츄얼과 앱스트랙트의 차이는 버츄얼은 자손이 재정의를 할수도있고 안할수도 있으니 코드를 한줄이라도 짜놔야 하는데 추상은 반드시 재정의를 해야하기 때문에 정의만 해도됨

메서드 : call by value : 메소드 호출 시 데이터를 복사해 매개 변수에 전달
           call by ref : 메소드 호출 시 직접 원본 변수의 값을 바꾸는 방법 (ref 키워드) 
           out : 컴파일러를 통해 결과를 할당하지 않는 버그 가능성 제거 (out 키워드, 권장) (출력전용에 의한 전달)
